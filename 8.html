Responsive Design

What does that mean?

(according to Ethan Marcotte)

A flexible grid (with flexible images) that incorporates media queries to create a responsive, adaptive layout.

Responsive design means:

  • Design for capabilities

  • Not individual browsers

  • Designing to screen size

  • not exact pixel dimensions

  • Designing for changing content

  • Hey, we should be doing that already!

Also, it’s kinda hard.

Why it’s hard

  • More configurations to test

  • Need to handle different vertical sizes and relations.

  • Images and media need to resize.

  • Titles and Menus need multiple formats.

  • Must have buy-in from clients, designers, and developers

  • Relatively new, libraries and support still works-in-progress

Why it’s awesome

  • Write once, display anywhere

  • Not designing for a static canvas, designing for responsiveness to content and the user.

  • Mobile support baked-in

  • True responsive design will display correctly on the iPad 23 in 2025.

  • Works regardless of device, orientation or resolution.

Flexible/Fluid Grids

  • Size everything in ems or percentages

  • Fluid percentages for width

  • Height needs to be set to auto.

  • Golden Formula: target ÷ context = result

Maintaining Layouts

  • If you have to set a width - try to only do it on the container - use percentages inside

  • a max-width of 100% will make sure your page doesn’t break out of the viewpoint

  • don’t use height in px for anything with text inside

Bryan Rieger, “Rethinking the Mobile Web”

Media Queries

Media Types

  • @media screen{...}

  • @media print{...}

  • @media handheld{...}

ex.
@media screen and (min-width:600px) and (max-width:1200px)

Enter Media Queries

  • Can be used inline in stylesheets:

    @media screen and (max-width:500px) { ... }

  • Can be used as separate stylesheets:

    <link rel="stylesheet" media="screen and (max-width:500px)" href="example.css" />

  • Can be used with @import (discouraged)

    @import url(example.css) screen and (max-width:500px);

Using Heroku

  • Download and install the heroku toolbelt

  • Create a repository on GitHub
